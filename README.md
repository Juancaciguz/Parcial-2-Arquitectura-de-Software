¿Cómo modificaría su diseño si este microservicio tuviera que comunicarse con otro servicio que almacena el historial de cálculos en una base de datos externa?
R/ Para permitir que el microservicio se comunique con un servicio externo de historial, lo más práctico sería usar comunicaciones asíncronas con un broker de mensajes como por ejemplo Kafka. La idea es que cuando nuestro microservicio termina de calcular el factorial y la paridad, no hable directamente con la base de datos del historial. En cambio, simplemente envía un mensaje con el numero, el factorial y la etiqueta a una cola usando una libreria de productor.
Luego, existe otro microservicio aparte, el microservicio de historial, que es el encargado de escuchar esa cola, recibir los mensajes y guardarlos en la base de datos externa. De esta forma, cada servicio se enfoca en lo suyo.
